## 구현 방안

- 2권 2장 "Nearby Friends" 에서 Redis Pub/Sub 파트만 Erlang/OTP로 대체 구현
- Erlang 활용 방안
  - 웹소켓 서버 → Erlang으로 구현
  - Redis Pub/Sub 클러스터 → 분산 Erlang 애플리케이션
  - 각 사용자는 Erlang 프로세스로 표현
- 아래 컴포넌트에 대한 구현은 무시
  - 위치 이동 이력 DB
  - 사용자 DB
  - HTTP API 서버

## 요구사항

- 400명의 친구를 Erlang 프로세스로 구현하며, 랜덤한 방향으로 랜덤한 속도로 이동
  - 400명의 친구들은 편의상 대한민국 내 랜덤하게 배치하고, 100명은 서울 내에 배치
  - 400명이 사용자의 친구 목록에 있는 것으로 가정하며, 이 이외의 별도 친구 관계 그래프는 필요 없음
  - 친구들이 이동할 수 있는 범위 또한 대한민국 안으로 한정
  - 랜덤 속도 기준: 도보 4km/h ~ 차량 60km/h 안에서 랜덤 설정
- 사용자가 8km 범위의 친구들의 위치를 검색한 이후 15초 간격으로 친구들 위치 갱신
  - WebSocket 연결을 통해 수신받아야 하며, 이는 Erlang 기반에서 구현
- React + Vite 조합으로 Naver Map 기반 UI 구현
- 가능하다면 위도, 경도를 직접 입력하지 않고 디바이스 위치에서 검색할 수 있도록 구현
- 사용자가 지도 상에서 특정 위치를 찍으면 해당 위치를 기반으로 8km 반경 친구 재검색

## 특이사항

- Erlang 패러다임을 제대로 학습할 수 있도록 Supervision Trees 기반의 OTP 애플리케이션 방식으로 구현할 것
- 가능하다면 실제 분산 Erlang 구조로 구현
- Naver Cloud Platform에서 발급받은 Client ID는 이미 있지만, 은닉하기 위해 `your-client-id` 같은 방식으로 입력해둘 것
